# CLAUDE.md — AI Assistant Guide for EV_page_profile

This file provides context and conventions for AI assistants working on this repository.

## Project Overview

**Static multi-page academic website** for Edoardo Vicentini (postdoctoral researcher, CIC nanoGUNE, Nanooptics Group). Published via GitHub Pages at `https://vicentini-edoardo.github.io/EV_page_profile/`.

**Stack:** Plain HTML5 + Vanilla CSS3 + Vanilla ES2015+ JavaScript — no framework, no bundler, no npm.

## Development Commands

```bash
# Local preview (serves at http://localhost:8000)
python3 -m http.server 8000

# Regenerate project detail pages after editing projects.json or the template
python3 scripts/generate_projects_pages.py

# Regenerate award image manifests after adding/removing award images
python3 scripts/generate_award_manifests.py

# Manually sync publications from OpenAlex
OPENALEX_API_KEY=<key> OPENALEX_MAILTO=you@example.com python3 scripts/update_publications_openalex.py
```

No build step, no `npm install`, no compilation. Changes to HTML/CSS/JS are immediately reflected on refresh.

## CI/CD

| `deploy.yml` | Push to `main` | Runs project page generator, uploads static artifact, deploys to GitHub Pages |

## Architecture & Key Conventions

### JavaScript Module Pattern

No bundler. Each module is an IIFE that exports to `window`:

```javascript
// core utility
(function() {
  window.DOM = { qs, qsa, on };
})();

// page module
(function() {
  window.PageModules = window.PageModules || {};
  window.PageModules.publications = { init };
})();
```

`main.js` dispatches to page modules based on the current page (via `document.body.dataset.page` or similar).

### Partial Injection

Header and footer are loaded client-side. JS fetches `assets/partials/header.html` and `assets/partials/footer.html` and injects them into `<div data-partial="header">` / `<div data-partial="footer">` markers. A custom `layout:ready` event fires after injection — page-specific code that depends on nav elements should listen for it.

### Site Metadata

Global site constants (name, affiliation, email, ORCID, CV URLs) live in `main.js` under a `SITE` object and are also available as `data-site-*` attributes on the `<html>` or `<body>` element for CSS/HTML-level access.

### CSS Layers (load order matters)

1. `tokens.css` — custom property definitions (colors, spacing, type scale, radius, shadows)
2. `base-core.css` — reset + base typography
3. `components.css` — reusable UI components
4. `pages.css` — page-specific overrides
5. `print.css` — `@media print` only

Always define new design values as tokens in `tokens.css`; never hardcode colors or spacing elsewhere.

### Key Design Tokens

```css
--accent: #5fd0e5       /* cyan, primary accent */
--accent-2: #f2b0a2     /* salmon, secondary accent */
--radius-sm: 8px
--radius-md: 14px
--radius-lg: 20px
--space-1 … --space-8   /* 0.25rem … 4rem */
--step--1 … --step-4    /* fluid font sizes via clamp() */
```

Font family: **IBM Plex** (Serif, Sans, Mono). Default theme: `data-theme="dark"` on `<html>`.

## Data Contracts

### `assets/data/projects.json`

Each project entry must include: `slug`, `title`, `summary`, `tags`, `status`, and optional `description`, `image`, `links`. The slug determines the generated filename (`projects/<slug>.html`). **Do not rename slugs** — that would break existing public URLs.

### `assets/data/publications.json`

Auto-generated by the OpenAlex sync script. **Do not edit manually.** To add a missing publication: add its DOI to `publications.selected.json` and re-run the sync.

### `assets/data/publication_images.json`

Maps DOI strings to image folder paths under `assets/img/publications/`. Format: `{ "10.xxxx/yyy": "folder-name" }`.

### `assets/data/awards.json` / `conference.json` / `funding.json`

Manually maintained. Preserve existing key schemas; do not add ad-hoc fields without updating the corresponding rendering code in `assets/js/pages/`.

## Coding Style

### HTML
- 2-space indentation
- Semantic elements (`nav`, `main`, `section`, `article`, `figure`)
- Data attributes for JS hooks: `data-partial`, `data-page`, `data-md`
- ARIA labels on interactive elements
- Keep `<link rel="canonical">` accurate per page

### CSS
- 2-space indentation
- All new values as CSS custom properties in `tokens.css`
- Mobile-first media queries
- BEM-like class naming: `.site-header`, `.hero__title`, `.card--featured`

### JavaScript
- 2-space indentation
- `const`/`let`, arrow functions, template literals
- IIFE modules exporting to `window.*` namespace
- No external libraries
- Null-check before DOM operations; use `?.` optional chaining
- Wrap `fetch()` calls with `AbortController` timeout
- No classes — prefer plain functions

### Python (scripts)
- PEP 8, 4-space indentation
- Use `argparse` or environment variables for configuration
- Always handle HTTP errors and missing data gracefully
- Print progress to stdout for CI log visibility

### Filenames
- All lowercase, hyphen-separated: `dual-comb-spectroscopy.html`
- Project slugs must match their filename (used in URL)

## Testing & Validation

No automated test suite. Manual validation process:

1. **Local preview**: `python3 -m http.server 8000`
2. **After JSON edits**: check the affected page renders correctly
3. **After `projects.json` or template changes**: run `generate_projects_pages.py`, verify a generated page
4. **After CSS/token changes**: check multiple pages in both dark and light themes, and in print preview
5. **After `publications.json` sync**: verify `publications.html` renders, filtering works, and citation chart loads
6. **After adding award images**: run `generate_award_manifests.py`, verify slideshow on `awards.html`
7. **Before merging**: open `index.html`, `research.html`, `publications.html`, `cv.html`, and at least one project detail page

---

## URL Stability

Public URLs are indexed. **Never rename or remove** a page without adding a redirect stub. Existing stubs: `talks.html` → `conference.html`, `projects/index.html` → `/projects.html`.

## Out of Scope / Anti-patterns

- Do not introduce a JavaScript framework (React, Vue, etc.)
- Do not add a package.json or bundler
- Do not add a CSS preprocessor (Sass, Less)
- Do not add automated tests without agreeing on a testing strategy first
- Do not hardcode colors, spacing, or font sizes outside `tokens.css`
- Do not commit `.DS_Store`, `_site/`, or `_archive_unused/` contents (all in `.gitignore`)
